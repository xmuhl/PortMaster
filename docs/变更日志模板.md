# PortMaster 变更日志

## 📋 变更日志说明

**文档目的**: 记录PortMaster项目的所有重要变更，包括功能添加、bug修复、架构调整等  
**版本规范**: 采用语义化版本控制 (Semantic Versioning)  
**格式标准**: 遵循 [Keep a Changelog](https://keepachangelog.com/) 规范  

## 🏷️ 版本标记说明

### 版本号格式: X.Y.Z
- **X (主版本号)**: 不兼容的API修改
- **Y (次版本号)**: 向下兼容的功能性新增
- **Z (修订版本号)**: 向下兼容的问题修正

### 变更类型
- **Added**: 新增功能
- **Changed**: 对现有功能的变更
- **Deprecated**: 不推荐使用的功能
- **Removed**: 移除的功能
- **Fixed**: 问题修复
- **Security**: 安全相关的修复

## 🔍 变更审查流程

### 变更提交要求
1. **代码变更**: 必须通过编译验证 (0 error 0 warning)
2. **功能变更**: 需要更新相关文档
3. **接口变更**: 需要更新API文档
4. **UI变更**: 需要截图对比验证

### 变更记录规范
1. **及时记录**: 每次重要变更都要及时更新日志
2. **详细描述**: 清楚说明变更的内容和影响
3. **关联引用**: 关联相关的问题编号或需求编号
4. **影响评估**: 说明变更对其他模块的影响
